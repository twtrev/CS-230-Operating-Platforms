# CS 230 Operating Platforms

## Course Overview
This repository contains coursework completed for CS 230: Operating Platforms at Southern New Hampshire University. The course focused on software design, operating system architectures, memory and storage management, distributed systems, networking, and security considerations for modern applications.

## Project: The Gaming Room – Draw It or Lose It
The primary project in this course involved creating a software design document for The Gaming Room's application, Draw It or Lose It. The goal was to evaluate different operating platforms and recommend a solution that would support expansion across multiple devices and operating systems.

## Skills Demonstrated
- Software design and analysis
- Operating system architecture evaluation
- Memory and storage management concepts
- Distributed systems and networking
- Security planning and user data protection
- Technical documentation and recommendations

## Reflection
Through this project, I gained a better understanding of how operating systems manage resources and support software applications. I learned how to evaluate different platforms, identify storage and memory management techniques, and recommend solutions that balance performance, scalability, and security. This course also strengthened my ability to communicate technical concepts through professional documentation.

## Portfolio Reflection

The client for this project was The Gaming Room, which wanted to expand its game, Draw It or Lose It, from an Android application to a web-based application that could support multiple operating systems. The software needed to allow multiple teams and players while ensuring that only one game instance could exist. The application also needed to be secure, scalable, and capable of supporting future growth.

One area I feel I did well was evaluating different operating platforms and recommending a Linux-based cloud environment that best met the client's requirements. Working through the software design document before writing code helped me better understand the system requirements and organize the application's architecture. If I could improve one part of my work, I would include more detailed diagrams and explanations of how the different software components interact to make the design easier for future developers to understand.

Throughout the project, I focused on the client's needs by designing a solution that emphasized scalability, security, and maintainability. Considering the user's needs is important because software should solve the client's problems while remaining reliable and easy to use. This project also reinforced the importance of planning before coding. In future software projects, I will continue using software design documents, UML diagrams, and platform evaluations to analyze requirements and develop more organized, efficient, and maintainable applications.
